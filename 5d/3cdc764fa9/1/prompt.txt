## Deviations from the design doc

1. **`add_to_draft_pool` keeps the v1 signature** (no `source` kwarg) — §4 wrote `add_to_draft_pool(...)` ambiguously; v1 parity minimizes churn. `LiveItem.to_dict()` omits empty `source` so migrated v1 draft-pool items (which had no source field) don't gain `source: ''`.
2. **Closed-archive filename↔id check added** (`closed/&lt;name&gt;.yaml: filename stem must equal sprint id …`) — not explicitly in §5, but carryover linearity depends on filename order, so a mismatch must surface.
3. **Migration guards apply to `--dry-run` too** — §7 specified the guards for `--apply`; a dry-run with no v1 ledger (or v2 present) also exits 2 since there's nothing to plan. This is what makes "second `--apply` exits 2" work (ledger.yaml is gone).
4. **`pause` writes the archive even with zero items** — preserves sprint history (v1 kept the closed record in the monolith).
5. **New `_stringify_dates` normalization** on outcome dicts at parse time — YAML unquoted dates became `datetime.date` objects and broke the JSON board (latent v1 bug, surfaced by the smoke test).
6. **`init_ledger` ordering**: existing state.yaml → `"sprint ledger already exists"` (v1 message); only-v1-ledger → the v1-detected message.
7. **`sprint status`** keeps v1's `Tasks:`/`Planning:` grouping for open items (design only said "open items"), adding the `Done (awaiting rollover):` section.
