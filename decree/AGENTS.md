# decree/AGENTS.md

Applies to dogfood PRD/ADR/SPEC documents under `decree/`.

These files are product truth for this repository. Treat them as structured
source, not free-form notes.

## Required Workflow

- Prefer `uv run decree new <type> "Title"` for new documents.
- Prefer `uv run decree status <ID> <action>` for lifecycle transitions.
- Run `uv run decree lint` after any document change.
- Run `uv run decree progress --doc <ID>` when checking acceptance criteria.
- Run `uv run decree index rebuild && uv run decree index verify` after
  frontmatter or `governs:` changes.

## Identity Rules

- Every document must have frontmatter `id: TYPE-ULID`.
- File names must start with the lowercase ID.
- Do not create new numeric IDs.
- Do not manually convert legacy IDs; use `decree migrate ids`.

## Generated Files

- `decree/*/index.md` is generated by `decree index regenerate`.
- `decree/*/reports/*.md` are report snapshots generated by
  `decree report regenerate`.
- Do not manually edit generated sections unless you are changing generator
  behavior and tests.

## Acceptance Criteria

- Primary acceptance criteria count toward progress.
- Deferred/out-of-scope sections are counted separately.
- Do not mark a SPEC or PRD implemented until primary criteria are complete.
- If code is complete but the SPEC is not updated, update the SPEC rather than
  leaving progress stale.

## Governance Fields

- `references:` links PRD/ADR/SPEC decision chains.
- `governs:` links decisions to repo-relative files or directories.
- Use `uv run decree why <path>` to verify ownership.
- Missing governance is an explicit gap; do not invent ownership in summaries.
