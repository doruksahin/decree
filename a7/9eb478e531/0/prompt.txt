Review this proposed sprint mechanism for Decree as a product/architecture design, not implementation. Do not edit files. Be skeptical and look for drift risks, missing invariants, confusing edge cases, and command semantics that could cause bad history. Context: Decree tracks PRD/ADR/SPEC decision chains. PRD/ADR are context, SPEC is the main executable sprint task. New SPEC defaults into active sprint; explicit backlog allowed. Same PRD can have SPECs across different sprints. Same SPEC can move to a later sprint only as explicit carryover during sprint rollover with required reason. Closed sprints are immutable. Sprint finish requires each incomplete task to be done/carryover/defer/drop with reason. Lint errors include no active sprint, multiple active sprints, incomplete closed task without outcome, same SPEC in multiple sprints without carryover link, open SPEC neither active nor backlog, and closed sprint mutated.
